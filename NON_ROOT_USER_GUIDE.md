# 🔐 Non-Root User Security System Guide

## Overview

The Non-Root User Security System implements comprehensive container security by ensuring all containers run as non-privileged users, following security best practices and compliance standards.

## 🎯 Key Features

### Security Hardening
- **Non-root execution** - All containers run as unprivileged users (UID 1001)
- **Capability dropping** - Remove unnecessary Linux capabilities
- **Read-only filesystems** - Prevent runtime modifications
- **Privilege escalation prevention** - Block privilege escalation attempts
- **Resource isolation** - Isolate container resources and namespaces

### Compliance Frameworks
- **CIS Docker Benchmark** - Industry standard container security
- **NIST 800-190** - Container security guidelines
- **PCI DSS** - Payment card industry security standards  
- **SOX** - Sarbanes-Oxley compliance requirements

### Integration Features
- **Multi-system integration** - Works with existing security systems
- **Automated enforcement** - Continuous policy enforcement
- **Real-time monitoring** - Live security status monitoring
- **Alert generation** - Immediate notification of security issues
- **Compliance reporting** - Automated compliance documentation

## 🐳 Container Implementation

### Secure Dockerfile Example
```dockerfile
# Secure Dockerfile for trading-bot
# Generated by Non-Root Integration System
FROM python:3.11-alpine

# Security and compliance labels
LABEL security.non_root="true"
LABEL security.compliance="cis-docker-benchmark,nist-800-190"
LABEL security.level="high"
LABEL security.scan="enabled"
LABEL maintainer="AI Trading Bot Security System"
LABEL version="1.0"

# Create non-root group and user
RUN addgroup -g 1001 -S appgroup && \
    adduser -u 1001 -S appuser -G appgroup

# Create application directories with proper ownership
RUN mkdir -p /app \
    /app/logs \
    /app/data \
    /app/config && \
    chown -R 1001:1001 /app && \
    chmod -R 755 /app

# Set specific directory permissions
RUN chmod 755 /app/logs && \
    chmod 644 /app/data && \
    chmod 600 /app/config

# Copy application files with proper ownership
COPY --chown=1001:1001 . /app/

# Set working directory
WORKDIR /app

# Python application setup
ENV PYTHONUNBUFFERED=1 \
    PYTHONDONTWRITEBYTECODE=1 \
    PIP_NO_CACHE_DIR=1 \
    PIP_DISABLE_PIP_VERSION_CHECK=1 \
    PYTHONPATH=/app

# Install Python dependencies as non-root user
COPY --chown=1001:1001 requirements.txt /tmp/
RUN pip install --user --no-cache-dir -r /tmp/requirements.txt && \
    rm /tmp/requirements.txt

# Add user site-packages to PATH
ENV PATH=/home/appuser/.local/bin:$PATH

# Additional security hardening
RUN find /app -type f -name '*.py' -exec chmod 644 {} \; 2>/dev/null || true
RUN find /app -type d -exec chmod 755 {} \; 2>/dev/null || true

# Health check for container monitoring
HEALTHCHECK --interval=30s --timeout=10s --start-period=5s --retries=3 \
    CMD python -c "import requests; requests.get('http://localhost:5001/health', timeout=5)" || exit 1

# Switch to non-root user for security
USER 1001:1001

# Security labels
LABEL security.non_root="true"
LABEL security.user="appuser"
LABEL security.uid="1001"
LABEL security.gid="1001"
LABEL security.compliance="cis-docker-benchmark"

# Default command
CMD ["python", "main.py"]
```

## 🛡️ Security Best Practices

### 1. User and Permission Management
```bash
# Verify non-root execution
docker exec container-name whoami  # Should NOT return 'root'

# Check file permissions
docker exec container-name ls -la /app

# Verify user ID
docker exec container-name id
```

### 2. Capability Management
```bash
# Check dropped capabilities
docker inspect container-name | jq '.[0].HostConfig.CapDrop'

# Verify no privileged mode
docker inspect container-name | jq '.[0].HostConfig.Privileged'
```

### 3. Filesystem Security
```bash
# Verify read-only root filesystem
docker inspect container-name | jq '.[0].HostConfig.ReadonlyRootfs'

# Check tmpfs mounts
docker inspect container-name | jq '.[0].HostConfig.Tmpfs'
```

### 4. Network Security
```bash
# Verify network isolation
docker network ls
docker network inspect secure-trading-network

# Check port exposure
docker port container-name
```

## 🚨 Troubleshooting

### Common Issues

#### 1. Permission Denied Errors
```bash
# Problem: Application can't write to directories
# Solution: Ensure proper ownership and permissions

# Check ownership
docker exec container-name ls -la /app

# Fix ownership (in Dockerfile)
COPY --chown=1001:1001 . /app/
RUN chown -R 1001:1001 /app
```

#### 2. Package Installation Issues
```bash
# Problem: Can't install packages as non-root user
# Solution: Use --user flag or install before USER directive

# Install as user
RUN pip install --user package-name

# Or install before switching user
RUN pip install package-name
USER 1001:1001
```

#### 3. Health Check Failures
```bash
# Problem: Health checks fail due to permissions
# Solution: Ensure health check runs as correct user

# Check health check command
HEALTHCHECK --interval=30s --timeout=10s \
    CMD python -c "import requests; requests.get('http://localhost:5001/health')"
```

#### 4. Volume Mount Issues
```bash
# Problem: Volume permissions don't match container user
# Solution: Set proper volume ownership

# Using init containers or proper ownership
volumes:
  - ./logs:/app/logs
# Ensure host directory has proper permissions
sudo chown -R 1001:1001 ./logs
```

## 📈 Performance Impact

### Security vs Performance
- **Minimal overhead** - Non-root execution adds negligible performance impact
- **Resource efficiency** - Capability dropping reduces resource usage
- **Startup time** - Slight increase due to permission setup (~1-2 seconds)
- **Runtime performance** - No measurable impact on application performance

### Optimization Tips
```dockerfile
# Combine RUN commands to reduce layers
RUN addgroup -g 1001 -S appgroup && \
    adduser -u 1001 -S appuser -G appgroup && \
    mkdir -p /app && \
    chown -R 1001:1001 /app

# Use multi-stage builds for smaller images
FROM python:3.11-alpine AS builder
# Build stage...

FROM python:3.11-alpine AS runtime
# Runtime stage with non-root user...
```
